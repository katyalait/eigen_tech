8""" 

9A collection of methods for tree (grammar) transformations used 

10in parsing natural language. 

11 

12Although many of these methods are technically grammar transformations 

13(ie. Chomsky Norm Form), when working with treebanks it is much more 

14natural to visualize these modifications in a tree structure. Hence, 

15we will do all transformation directly to the tree itself. 

16Transforming the tree directly also allows us to do parent annotation. 

17A grammar can then be simply induced from the modified tree. 

18 

19The following is a short tutorial on the available transformations. 

20 

21 1. Chomsky Normal Form (binarization) 

22 

23 It is well known that any grammar has a Chomsky Normal Form (CNF) 

24 equivalent grammar where CNF is defined by every production having 

25 either two non-terminals or one terminal on its right hand side. 

26 When we have hierarchically structured data (ie. a treebank), it is 

27 natural to view this in terms of productions where the root of every 

28 subtree is the head (left hand side) of the production and all of 

29 its children are the right hand side constituents. In order to 

30 convert a tree into CNF, we simply need to ensure that every subtree 

31 has either two subtrees as children (binarization), or one leaf node 

32 (non-terminal). In order to binarize a subtree with more than two 

33 children, we must introduce artificial nodes. 

34 

35 There are two popular methods to convert a tree into CNF: left 

36 factoring and right factoring. The following example demonstrates 

37 the difference between them. Example:: 

38 

39 Original Right-Factored Left-Factored 

40 

41 A A A 

42 / | \ / \ / \ 

43 B C D ==> B A|<C-D> OR A|<B-C> D 

44 / \ / \ 

45 C D B C 

46 

47 2. Parent Annotation 

48 

49 In addition to binarizing the tree, there are two standard 

50 modifications to node labels we can do in the same traversal: parent 

51 annotation and Markov order-N smoothing (or sibling smoothing). 

52 

53 The purpose of parent annotation is to refine the probabilities of 

54 productions by adding a small amount of context. With this simple 

55 addition, a CYK (inside-outside, dynamic programming chart parse) 

56 can improve from 74% to 79% accuracy. A natural generalization from 

57 parent annotation is to grandparent annotation and beyond. The 

58 tradeoff becomes accuracy gain vs. computational complexity. We 

59 must also keep in mind data sparcity issues. Example:: 

60 

61 Original Parent Annotation 

62 

63 A A^<?> 

64 / | \ / \ 

65 B C D ==> B^<A> A|<C-D>^<?> where ? is the 

66 / \ parent of A 

67 C^<A> D^<A> 

68 

69 

70 3. Markov order-N smoothing 

71 

72 Markov smoothing combats data sparcity issues as well as decreasing 

73 computational requirements by limiting the number of children 

74 included in artificial nodes. In practice, most people use an order 

75 2 grammar. Example:: 

76 

77 Original No Smoothing Markov order 1 Markov order 2 etc. 

78 

79 __A__ A A A 

80 / /|\ \ / \ / \ / \ 

81 B C D E F ==> B A|<C-D-E-F> ==> B A|<C> ==> B A|<C-D> 

82 / \ / \ / \ 

83 C ... C ... C ... 

84 

85 

86 

87 Annotation decisions can be thought about in the vertical direction 

88 (parent, grandparent, etc) and the horizontal direction (number of 

89 siblings to keep). Parameters to the following functions specify 

90 these values. For more information see: 

91 

92 Dan Klein and Chris Manning (2003) "Accurate Unlexicalized 

93 Parsing", ACL-03. http://www.aclweb.org/anthology/P03-1054 

94 

95 4. Unary Collapsing 

96 

97 Collapse unary productions (ie. subtrees with a single child) into a 

98 new non-terminal (Tree node). This is useful when working with 

99 algorithms that do not allow unary productions, yet you do not wish 

100 to lose the parent information. Example:: 

101 

102 A 

103 | 

104 B ==> A+B 

105 / \ / \ 

106 C D C D 

107 

108"""

114def chomsky_normal_form( 

115 tree, factor="right", horzMarkov=None, vertMarkov=0, childChar="|", parentChar="^" 

116): 

117 # assume all subtrees have homogeneous children 

118 # assume all terminals have no siblings 

119 

120 # A semi-hack to have elegant looking code below. As a result, 

121 # any subtree with a branching factor greater than 999 will be incorrectly truncated. 

122 if horzMarkov is None: 

123 horzMarkov = 999 

124 

125 # Traverse the tree depth-first keeping a list of ancestor nodes to the root. 

126 # I chose not to use the tree.treepositions() method since it requires 

127 # two traversals of the tree (one to get the positions, one to iterate 

128 # over them) and node access time is proportional to the height of the node. 

129 # This method is 7x faster which helps when parsing 40,000 sentences. 

130 

131 nodeList = [(tree, [tree.label()])] 

132 while nodeList != []: 

133 node, parent = nodeList.pop() 

134 if isinstance(node, Tree): 

135 

136 # parent annotation 

137 parentString = "" 

138 originalNode = node.label() 

139 if vertMarkov != 0 and node != tree and isinstance(node[0], Tree): 

140 parentString = "%s<%s>" % (parentChar, "-".join(parent)) 

141 node.set_label(node.label() + parentString) 

142 parent = [originalNode] + parent[: vertMarkov - 1] 

143 

144 # add children to the agenda before we mess with them 

145 for child in node: 

146 nodeList.append((child, parent)) 

147 

148 # chomsky normal form factorization 

149 if len(node) > 2: 

150 childNodes = [child.label() for child in node] 

151 nodeCopy = node.copy() 

152 node[0:] = [] # delete the children 

153 

154 curNode = node 

155 numChildren = len(nodeCopy) 

156 for i in range(1, numChildren - 1): 

157 if factor == "right": 

158 newHead = "%s%s<%s>%s" % ( 

159 originalNode, 

160 childChar, 

161 "-".join( 

162 childNodes[i : min([i + horzMarkov, numChildren])] 

163 ), 

164 parentString, 

165 ) # create new head 

166 newNode = Tree(newHead, []) 

167 curNode[0:] = [nodeCopy.pop(0), newNode] 

168 else: 

169 newHead = "%s%s<%s>%s" % ( 

170 originalNode, 

171 childChar, 

172 "-".join( 

173 childNodes[max([numChildren - i - horzMarkov, 0]) : -i] 

174 ), 

175 parentString, 

176 ) 

177 newNode = Tree(newHead, []) 

178 curNode[0:] = [newNode, nodeCopy.pop()] 

179 

180 curNode = newNode 

181 

182 curNode[0:] = [child for child in nodeCopy]

185def un_chomsky_normal_form( 

186 tree, expandUnary=True, childChar="|", parentChar="^", unaryChar="+" 

187): 

188 # Traverse the tree-depth first keeping a pointer to the parent for modification purposes. 

189 nodeList = [(tree, [])] 

190 while nodeList != []: 

191 node, parent = nodeList.pop() 

192 if isinstance(node, Tree): 

193 # if the node contains the 'childChar' character it means that 

194 # it is an artificial node and can be removed, although we still need 

195 # to move its children to its parent 

196 childIndex = node.label().find(childChar) 

197 if childIndex != -1: 

198 nodeIndex = parent.index(node) 

199 parent.remove(parent[nodeIndex]) 

200 # Generated node was on the left if the nodeIndex is 0 which 

201 # means the grammar was left factored. We must insert the children 

202 # at the beginning of the parent's children 

203 if nodeIndex == 0: 

204 parent.insert(0, node[0]) 

205 parent.insert(1, node[1]) 

206 else: 

207 parent.extend([node[0], node[1]]) 

208 

209 # parent is now the current node so the children of parent will be added to the agenda 

210 node = parent 

211 else: 

212 parentIndex = node.label().find(parentChar) 

213 if parentIndex != -1: 

214 # strip the node name of the parent annotation 

215 node.set_label(node.label()[:parentIndex]) 

216 

217 # expand collapsed unary productions 

218 if expandUnary == True: 

219 unaryIndex = node.label().find(unaryChar) 

220 if unaryIndex != -1: 

221 newNode = Tree( 

222 node.label()[unaryIndex + 1 :], [i for i in node] 

223 ) 

224 node.set_label(node.label()[:unaryIndex]) 

225 node[0:] = [newNode] 

226 

227 for child in node: 

228 nodeList.append((child, node))

231def collapse_unary(tree, collapsePOS=False, collapseRoot=False, joinChar="+"): 

232 """ 

233 Collapse subtrees with a single child (ie. unary productions) 

234 into a new non-terminal (Tree node) joined by 'joinChar'. 

235 This is useful when working with algorithms that do not allow 

236 unary productions, and completely removing the unary productions 

237 would require loss of useful information. The Tree is modified 

238 directly (since it is passed by reference) and no value is returned. 

239 

240 :param tree: The Tree to be collapsed 

241 :type tree: Tree 

242 :param collapsePOS: 'False' (default) will not collapse the parent of leaf nodes (ie. 

243 Part-of-Speech tags) since they are always unary productions 

244 :type collapsePOS: bool 

245 :param collapseRoot: 'False' (default) will not modify the root production 

246 if it is unary. For the Penn WSJ treebank corpus, this corresponds 

247 to the TOP -> productions. 

248 :type collapseRoot: bool 

249 :param joinChar: A string used to connect collapsed node values (default = "+") 

250 :type joinChar: str 

251 """ 

252 

253 if collapseRoot == False and isinstance(tree, Tree) and len(tree) == 1: 

254 nodeList = [tree[0]] 

255 else: 

256 nodeList = [tree] 

257 

258 # depth-first traversal of tree 

259 while nodeList != []: 

260 node = nodeList.pop() 

261 if isinstance(node, Tree): 

262 if ( 

263 len(node) == 1 

264 and isinstance(node[0], Tree) 

265 and (collapsePOS == True or isinstance(node[0, 0], Tree)) 

266 ): 

267 node.set_label(node.label() + joinChar + node[0].label()) 

268 node[0:] = [child for child in node[0]] 

269 # since we assigned the child's children to the current node, 

270 # evaluate the current node again 

271 nodeList.append(node) 

272 else: 

273 for child in node: 

274 nodeList.append(child)

283 """ 

284 A demonstration showing how each tree transform can be used.

287 from nltk.draw.tree import draw_trees 

288 from nltk import tree, treetransforms 

289 from copy import deepcopy 

290 

291 # original tree from WSJ bracketed text 

292 sentence = """(TOP 

293 (S 

294 (S 

295 (VP 

296 (VBN Turned) 

297 (ADVP (RB loose)) 

298 (PP 

299 (IN in) 

300 (NP 

301 (NP (NNP Shane) (NNP Longman) (POS 's)) 

302 (NN trading) 

303 (NN room))))) 

304 (, ,) 

305 (NP (DT the) (NN yuppie) (NNS dealers)) 

306 (VP (AUX do) (NP (NP (RB little)) (ADJP (RB right)))) 

307 (. .)))""" 

308 t = tree.Tree.fromstring(sentence, remove_empty_top_bracketing=True) 

309 

310 # collapse subtrees with only one child 

311 collapsedTree = deepcopy(t) 

312 treetransforms.collapse_unary(collapsedTree) 

313 

314 # convert the tree to CNF 

315 cnfTree = deepcopy(collapsedTree) 

316 treetransforms.chomsky_normal_form(cnfTree) 

317 

318 # convert the tree to CNF with parent annotation (one level) and horizontal smoothing of order two 

319 parentTree = deepcopy(collapsedTree) 

320 treetransforms.chomsky_normal_form(parentTree, horzMarkov=2, vertMarkov=1) 

321 

322 # convert the tree back to its original form (used to make CYK results comparable) 

323 original = deepcopy(parentTree) 

324 treetransforms.un_chomsky_normal_form(original) 

325 

326 # convert tree back to bracketed text 

327 sentence2 = original.pprint() 

328 print(sentence) 

329 print(sentence2) 

330 print("Sentences the same? ", sentence == sentence2) 

331 

332 draw_trees(t, collapsedTree, cnfTree, parentTree, original)
